## Root Cause Analysis — Why Our Numbers Diverge

### Before Fixes: Calibration Gaps (2026-06-03)

| Metric         | Our Calc   | Pradeep    | Gap         | Root Cause                                                                       |
| -------------- | ---------- | ---------- | ----------- | -------------------------------------------------------------------------------- |
| Universe       | 6,564      | 6,462      | +102        | Close — slight filter difference                                                 |
| Breakout       | 111        | 153        | -42         | Minor — volume filter?                                                           |
| Breakdown      | 438        | 431        | +7          | ✅ Near exact                                                                     |
| Monthly 25↑    | 234        | 215        | +19         | ✅ Close                                                                          |
| **Primary Up** | **593**    | **1,469**  | **-876**    | **① ADRs excluded + ② Unclassified tickers dropped + ③ Missing delisted stocks** |
| **T2108**      | **61.93%** | **39.03%** | **+22.9pp** | **① Denominator too small (only ≥40d tickers) + ② Missing delisted stocks**      |
| **Fib 13↑**    | **821**    | **1,581**  | **-760**    | Same as Primary Up                                                               |

### Three Root Causes Identified

#### Root Cause #1: ADRs Excluded (FIXED)

Our `ETF_KEYWORDS` in `db.py` included `" ADR"`, `" Depositary"`, `" American Depositary"`. Pradeep **includes ADRs** in his standard combo list.

**Impact:** ~300–500 ADR tickers silently dropped from our universe.

**Fix applied:** Removed ADR keywords from `ETF_KEYWORDS`. Need to run `db.reclassify_universe_etfs()` to update existing rows.

#### Root Cause #2: T2108 Denominator Wrong (FIXED)

Our `breadth_calculator.py` used `eligible = metrics["above_ma40"].notna().sum()` as denominator — only tickers with ≥40 days of history. Pradeep's denominator is the **full universe count** (~6,462).

**Impact:** Narrower denominator → artificially higher T2108. Stocks without 40 days of data should count as "not above MA40" in the denominator.

**Fix applied:** Changed denominator to `broad_count` (full broad universe count).

#### Root Cause #3: Missing Delisted Stocks (PIPELINE BUILT, NOT YET RUN)

Our FMP screener only returns **active** tickers. Delisted/bankrupt stocks don't appear, so we never downloaded their historical data. This means:

- **Primary Up is low:** We miss stocks that were up 25%+ but got delisted during the lookback
- **Primary Down is low:** We miss stocks that crashed and got delisted
- **T2108 is high:** Delisted stocks (mostly below MA40) don't count in denominator → % above inflates

**FMP provides a solution:** The `/stable/delisted-companies` API (verified working 2026-06-05). Historical prices available for **select US companies** — coverage is partial but significant.

**Pipeline built:** `delisted_loader.py` — fetches delisted list → adds to universe → backfills historical data.
