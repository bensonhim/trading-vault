## Exact TC2000 Formulas (Verbatim from the Video)

### 1. 4% BREAKOUT (Daily Intensity — Up)
```
(100 * (C - C1) / C1) >= 4 
AND V >= 100000 
AND V > V1
```
**Simplified:** `C/C1 - 1 >= 0.04` = C/C1 >= 1.04
**Additional filters:**
- `V >= 100000` — today's volume >= 100,000 shares
- `V > V1` — volume greater than yesterday (optional refinement)

---

### 2. 4% BREAKDOWN (Daily Intensity — Down)
```
C / C1 <= .96 
AND V >= 100000 
AND V > V1
```
**Simplified:** `C/C1 <= 0.96`
**Additional filters:** Same as breakout

---

### 3. PRIMARY UP (Quarterly Momentum)
```
100 * ((C + .01) - (MINC65 + .01)) / (MINC65 + .01) >= 25 
and AVGC20 * AVGV20 >= 250000
```
**Simplified:** `(C - MINC65) / MINC65 >= 0.25`
**Key terms:**
- `MINC65` = **minimum close over last 65 trading days** (NOT close 65 days ago)
- `.01` padding = avoids division by zero when MINC65 approaches 0
- `AVGC20 * AVGV20 >= 250000` = 20-day average close × 20-day average volume >= $250,000 **daily dollar turnover filter**

**💡 CRITICAL DIFFERENCE:** This formula measures stocks that are **up 25% from their 65-day LOW**. It does **NOT** compare today vs 65 days ago. 

Example:
- Stock went: $100 → $50 (low) → $65 (today)
- Our code (C/C65): $65/$100 − 1 = **−35%** → Primary Down ✓
- Pradeep's formula (MINC65): $65/$50 − 1 = **+30%** → Primary Up ✓

A stock that rallied 30% from its low but is still down from 65 days ago gets counted as **Primary Up** by Pradeep.

**This explains the 2.5x per-stock qualification rate gap.**

---

### 4. PRIMARY DOWN (Quarterly Momentum)
```
(100 * ((C + .01) - (MAXC65 + .01)) / (MAXC65 + .01)) <= (-25) 
and AVGC20 * AVGV20 >= 250000
```
**Simplified:** `(C - MAXC65) / MAXC65 <= −0.25`
**Key terms:**
- `MAXC65` = **maximum close over last 65 trading days**
- This measures stocks that are **down 25% from their 65-day HIGH**

**Both Primary Up and Primary Down can be TRUE for the same stock** if it made both a +25% rally from the low AND a −25% decline from the high during the quarter. They are **not mutually exclusive.**

---

### 5. MONTHLY 25 UP (Monthly Momentum)
```
C20 >= 5 
AND (AVGC20 * AVGV20) >= 250000 
AND 100 * (C - C20) / C20 >= 25
```
**Simplified:**
- `C20 >= 5` — price was ≥ $5 **22 trading days ago** (C20 = close 20 bars ago, wait — no, C20 means close 20 bars ago, but for "monthly" Pradeep expects 22 trading days... hmm)
- `AVGC20 * AVGV20 >= 250000` — same daily dollar turnover filter
- `(C - C20) / C20 >= 0.25` = `C/C20 >= 1.25` — up 25% in ~22 trading days

**Question:** Why C20 and not C22? Video says "month" but uses 20 bars. This might be intentional (20 trading days ≈ 1 calendar month).

### 9. FIB 13 UP (Fibonacci-Inspired Half-Quarter)
```
100 * ((C + .01) - (MINC34 + .01)) / (MINC34 + .01) >= 13 
and (AVGC20 * AVGV20) >= 250000
```
**Simplified:** `(C - MINC34) / MINC34 >= 0.13`
- Uses `MINC34` (NOT C34!) — same pattern as Primary but with 34-day lookback
- 13% gain from the 34-day low

## Differences from Our Previous Implementation

| Dimension | Our Code | Pradeep's Formula | Impact |
|-----------|----------|------------------|--------|
| **Primary Up base** | Close 65 days ago (`C65`) | **65-day LOW** (`MINC65`) | Many stocks that had rallies within the quarter but didn't end up qualify for Pradeep | |
| **Primary Down base** | Close 65 days ago (`C65`) | **65-day HIGH** (`MAXC65`) | Many stocks that had crashes within the quarter but recovered slightly qualify for Pradeep | |
| **Breakout volume** | None | `V >= 100000 AND V > V1` | Our breakout counts might be slightly higher | |
| **Breadth dollar filter** | None | `AVGC20 * AVGV20 >= 250000` | All breadth metrics (Primary, Monthly, Fib) have a minimum dollar turnover filter! | |
| **Monthly lookback** | 22 days (`C22`) | 20 days (`C20`) | Minor off-by-2 | |
| **Monthly price floor** | None | `C20 >= 5` | Stocks that were below $5 at C20 excluded from monthly counts | |
| **Price type** | Raw (non-split-adjusted) | Split-adjusted (TC2000 default) | Corrupted returns for stocks with splits | |

### Step 3: Breadth Calculator Rewrite

```python
def calc_primary_up(ticker_df, target_date_str):
    """100 * ((C + .01) - (MINC65 + .01)) / (MINC65 + .01) >= 25"""
    c = ticker_df[ticker_df["date"] == target_date_str]["close"].iloc[0]
    minc65 = ticker_df.iloc[-65:]["close"].min()  # last 65 bars
    return ((c + 0.01) - (minc65 + 0.01)) / (minc65 + 0.01) >= 0.25

def calc_primary_down(ticker_df, target_date_str):
    """(100 * ((C + .01) - (MAXC65 + .01)) / (MAXC65 + .01)) <= -25"""
    c = ticker_df[ticker_df["date"] == target_date_str]["close"].iloc[0]
    maxc65 = ticker_df.iloc[-65:]["close"].max()
    return ((c + 0.01) - (maxc65 + 0.01)) / (maxc65 + 0.01) <= -0.25
```

### Step 4: Dollar Turnover Filter
- `AVGC20 * AVGV20 >= 250000` requires computing the 20-day average close and 20-day average volume
- This filter applies to **every indicator except breakout/breakdown** (which have their own V >= 100K filter)
- We can pre-compute this per ticker per date using a rolling window
