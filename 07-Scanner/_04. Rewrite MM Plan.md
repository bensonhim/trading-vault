# Rewrite MM Plan — From First Principles

> **Trigger:** Q2 2026 full-quarter comparison (`_q2_comparison.md`) revealed a **2.5x per-stock qualification rate gap** (our 6.2% vs Pradeep's 15.3% for Primary Up). This cannot be explained by missing stocks — the calculation logic itself is wrong.

---

## 🔴 Phase 0 COMPLETE: Exact Formulas Found

Pradeep's video "Introduction to Market Monitor" contains the **exact TC2000 PCF formulas**. These are completely different from our assumptions (C/C65 for Primary is WRONG — it's MINC65/MAXC65).

**The formulas are live in:** `Trading Radar Engine/_00. Indicator Definitions — Source of Truth.md`

### Key Formula Corrections

| Indicator | Our Old Formula (WRONG) | Pradeep's Real Formula | Impact |
|-----------|---------------------|---------------------|---------|
| **Primary Up** | `C/C65 >= 1.25` | `100*((C+.01)-(MINC65+.01))/(MINC65+.01) >= 25` | Uses 65-day **LOW**, not close 65 days ago |
| **Primary Down** | `C/C65 <= 0.75` | `100*((C+.01)-(MAXC65+.01))/(MAXC65+.01) <= -25` | Uses 65-day **HIGH**, not close 65 days ago |
| **Fib 13↑** | `C/C34 >= 1.13` | `100*((C+.01)-(MINC34+.01))/(MINC34+.01) >= 13` | Uses 34-day **LOW**, not close 34 days ago |
| **Fib 13↓** | `C/C34 <= 0.87` | `100*((C+.01)-(MAXC34+.01))/(MAXC34+.01) <= -13` | Uses 34-day **HIGH**, not close 34 days ago |
| **Monthly 25↑** | `C/C22 >= 1.25` | `100*(C-C20)/C20 >= 25` (with `C20 >= 5`) | Uses **C20**, not C22; adds price floor |
| **All Breadth** | No dollar filter | `AVGC20 * AVGV20 >= 250000` | Minimum **daily dollar turnover** ~$250K |
| **Data** | Raw (unadjusted) close | Split-adjusted close (TC2000 default) | Split effects corrupt 65-day returns |

### Why the 2.5x Gap Disappears

Pradeep's Primary Up formula `(C - MINC65)/MINC65 >= 0.25` measures **any stock that is up 25% from its 65-day low**. A stock that crashed 50% and rallied back 30% qualifies. Our formula `C/C65 >= 1.25` measures **only stocks that end the quarter above where they started**. This is a fundamentally different metric!

**This is the single biggest calibration discovery we've made.**

## Phase 1: Data Layer Rewrite

### 1.1 Switch to Split-Adjusted Close

**Problem:** `weekly_loader.py` uses `/stable/historical-price-eod/non-split-adjusted` endpoint and stores `adjClose` (which is the RAW closing price) in the `close` column.

**Why this breaks multi-day returns:** Any stock with a split in the last 65 days shows an artificial decline or gain. Example: NVDA 10:1 split (Jun 2024). Raw close went from ~$1200 to ~$120. Our 65-day return would show -90% decline. Split-adjusted: normal return.

**Fix:**
- Switch to `get_historical_daily()` which calls `/stable/historical-price-eod/full` (split-adjusted endpoint)
- Store `close` from that endpoint (which IS split-adjusted) in the `close` column
- Optionally keep `raw_close` from the NSA endpoint for display purposes

**Code Change (weekly_loader.py):**
```python
# BEFORE:
hist = fmp_adapter.get_historical_raw(ticker, from_date, to_date)
# ...
"close": row.get("adjClose") or row.get("close"),  # RAW close

# AFTER:
hist = fmp_adapter.get_historical_daily(ticker, from_date, to_date)
historical = hist.get("historical", []) if isinstance(hist, dict) else hist
# ...
"close": row.get("close"),  # Split-adjusted close
"raw_close": row.get("adjClose") or row.get("close"),  # Keep raw for reference
```

**DB Schema Change:**
- Add `raw_close REAL` column to `daily_ohlcv`
- `close` column becomes split-adjusted (used for all lookback calculations)
- `raw_close` stored for reference/display only

**Re-backfill required:** Yes. All 8,753 tickers × 1,450 days = ~3.5M API calls at Premium rate. Estimated 4-6 hours.

### 1.2 Remove Volume Filter from Universe Fetch

**Problem:** `weekly_loader.py:27-30` passes `volume_min=100000` to the screener. This excludes ~500-700 low-volume stocks that Pradeep includes in his broad MM universe.

**Why it matters:** Pradeep's Market Monitor uses ALL stocks with price ≥ $3 (no volume filter). Volume filter is only for setup scans (SOS, DEP, Sugar Babies).

**Fix:**
```python
# BEFORE:
def fetch_universe_tickers() -> list[str]:
    screener = fmp_adapter.get_stock_screener(
        country="US", price_min=MIN_PRICE, volume_min=MIN_VOLUME, limit=10000
    )

# AFTER:
def fetch_universe_tickers(volume_filter: bool = False) -> list[str]:
    params = {"country": "US", "price_min": MIN_PRICE, "limit": 10000}
    if volume_filter:
        params["volume_min"] = MIN_VOLUME
    screener = fmp_adapter.get_stock_screener(**params)
```

**Re-backfill required:** Yes. New tickers (previously excluded by volume filter) need to be downloaded.

## Phase 2: Calculation Rewrite

### 2.1 Rewrite Breadth Calculator — Explicit Per-Indicator Functions

Replace the monolithic `_compute_ticker_metrics()` with individual functions, each documented with the exact Pradeep definition:

```python
def calc_primary_up(close_today: float, close_65d_ago: float) -> bool:
    """C/C65 >= 1.25 — Stock up ≥25% in 65 trading days.
    Source: MM Guide Part 4, Part 1.
    Universe: All US common stocks (+ ADRs + ETFs if Phase 0 confirms)."""
    if close_65d_ago <= 0:
        return False
    return (close_today / close_65d_ago) >= 1.25

def calc_t2108_above_ma40(closes_40d: list[float]) -> bool:
    """% stocks where close > SMA(close, 40).
    SMA includes current bar (last 40 bars including today).
    Source: MM Guide Part 8."""
    if len(closes_40d) < 40:
        return False
    ma40 = sum(closes_40d) / 40
    return closes_40d[-1] > ma40
```
