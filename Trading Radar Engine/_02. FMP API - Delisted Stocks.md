**No, FMP does not only provide data for active stocks.** They have a dedicated **Delisted Companies API** that gives you information on companies removed from US exchanges (and some international ones).

However, there are important nuances about **what** data is available for delisted stocks.

### What FMP Provides for Delisted Stocks

| Data Type                  | Available?                  | Details                                                                 | Coverage                          | Free Tier? |
|---------------------------|-----------------------------|-------------------------------------------------------------------------|-----------------------------------|------------|
| **List of delisted companies** | Yes                        | Symbol, company name, exchange, IPO date, delisted date                | Comprehensive list               | Yes       |
| **Historical price data**     | Partial (select only)      | You can query normal historical endpoints with old tickers             | **Select US companies only**     | Limited   |
| **Fundamentals / Financials** | Limited                    | Available for some, but not guaranteed                                 | Partial                          | Limited   |
| **Full history for every delisted ticker** | No                      | Not all symbols have complete data                                     | Gaps exist (data source limits)  | No        |

**Key points from FMP’s own documentation:**

- They provide a list of delisted companies so you can track the full lifecycle of tickers and avoid **survivorship bias** in backtests.
- Historical price data exists for **select US companies** that have been delisted. Some older or less common tickers may return no data.
- If you need more complete historical coverage for delisted stocks, FMP recommends contacting their account managers (this is typically an enterprise-level request).

### Delisted Companies API (Recommended Endpoint)

**Stable endpoint:**
```
https://financialmodelingprep.com/stable/delisted-companies?page=0&limit=100&apikey=YOUR_API_KEY
```

**Parameters:**
- `page` — pagination (starts at 0)
- `limit` — max 100 per request

**Example response fields:**
- `symbol`
- `companyName`
- `exchange`
- `ipoDate`
- `delistedDate`

This endpoint is available on the **free plan**.

### How to Get Historical Prices for a Delisted Ticker

You can use the **same historical price endpoints** we discussed earlier (including the unadjusted/non-split-adjusted one):

```python
# Try the normal historical endpoint with the old ticker
url = f"https://financialmodelingprep.com/stable/historical-price-eod/non-split-adjusted?symbol=OLD_TICKER&from=2010-01-01&to=2023-12-31&apikey=YOUR_KEY"
```

- If it returns data → the symbol is supported (even though delisted).
- If it returns empty or an error → that particular delisted stock is not covered in FMP’s historical price database.

This is the practical way most users handle it.

### Why This Matters for Your Use Case (Market Analysis & Indicators)

Since you were asking about **unadjusted close** and technical indicators:

- For serious backtesting or long-term indicator analysis, you **must** account for delisted stocks. Ignoring them creates **survivorship bias** (your strategy only sees winners that survived, inflating performance).
- Using the Delisted Companies API + unadjusted prices gives you a much more realistic picture.
- Many delisted stocks went bankrupt or were acquired — their price history often shows sharp drops. Adjusted data can mask this; unadjusted data preserves the real traded prices.

### Recommended Workflow

1. Regularly pull the full list of active symbols + delisted symbols.
2. When building historical datasets or running backtests:
   - For each symbol in your universe, check if it’s still active.
   - If not, look it up in the Delisted API to get the exact `delistedDate`.
   - Query historical prices only up to the delisted date (or try the endpoint anyway).
3. Handle cases where no historical data exists gracefully in your code.

### Limitations to Be Aware Of

- Not 100% coverage of every delisted stock’s price history (common industry limitation).
- No built-in “reason for delisting” field (you’d need to cross-reference news around the `delistedDate`).
- Older delisted companies (pre-2000s or very obscure ones) are more likely to have gaps.
